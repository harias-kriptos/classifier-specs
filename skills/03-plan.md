# Skill 03: Plan + tareas atómicas

Use this skill once the spec (Skill 02) is approved and committed. Output: a `todo.md` with atomic tasks ready for TDD execution.

**Recommended model:** Sonnet 4.6 (this is structural decomposition; doesn't need Opus depth).

**Role:** Tech Lead — see `roles/tech-lead.md`

---

## Context loading

1. `roles/tech-lead.md`
2. The approved spec `specs/NNN-<slug>.md` (full content)
3. `context/classifier-v2/current-decisions.md`
4. `stacks/python-lambda/rules.md`

---

## Objective

Decompose the spec into **atomic tasks** that can be implemented one at a time via TDD. Each task must have a test step BEFORE the code step — if it doesn't, it's not atomic.

Output: `todo.md` ready to commit at the root of the product repo.

---

## Procedure

1. Read the spec. Restate the goal in one sentence to confirm understanding.
2. Identify the **vertical slices** — minimum behaviors that can be tested independently. Example for `tree-url-generator`:
   - Slice 1: body validation rejects malformed inputs
   - Slice 2: ID sanitization rejects path traversal
   - Slice 3: UUID generation produces unique tree_id
   - Slice 4: pre-signed URL is signed with all required headers
   - Slice 5: 200 response has the documented structure
3. For each slice, write a task block:
   ```
   - [ ] <slice description>
     - [ ] RED: write test `test_<behavior>` asserting <what>
     - [ ] GREEN: minimal implementation in `src/<module>.py`
     - [ ] REFACTOR: <cleanup if any, or "skip">
   ```
4. Order tasks by dependency (no task should depend on a later task).
5. If a task exceeds ~30 minutes of estimated work, split it.

---

## Required output

`todo.md` content, ready to commit to the product repo:

```markdown
# todo — Spec NNN: <slug>

> Generated by Skill 03. Each task is a TDD cycle: RED → GREEN → REFACTOR.
> Mark `[x]` as you finish. Open BLOCKED if stuck after 3 attempts.

## Vertical slice 1: <name>
- [ ] RED: test `test_<behavior>` — <what it asserts>
- [ ] GREEN: minimal impl in `src/<path>`
- [ ] REFACTOR: <cleanup or "skip">

## Vertical slice 2: <name>
...
```

End with: **Siguiente paso:** Skill 04 — TDD implementation.

---

## Operating rules

- No task may combine "write test AND write code AND refactor" — those are three separate commits.
- Order: data validation first, then happy path, then error paths, then refactors.
- Maximum 10 vertical slices per spec. If you need more, the spec is too big — go back to Skill 02.
